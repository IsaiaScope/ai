# Spawn lifecycle robustness lives in iso-spawn, not in each skill

Detecting when a spawned tab is *done*, *alive*, or *dead* — and recovering its result without catching a pre-final turn — is the hard, easy-to-get-wrong part of driving another terminal. iso-review had to build it (`rv_wait_finished`/`rv_confirm_started`/`rv_recover_settled`) because the native `herdr agent wait --status idle` hangs when status sticks on `working`, recovers stale turns under jsonl flush-lag, and can't tell a slow-but-working agent from a dead one. The same gaps live in the shared layer — `deliver.sh:48` uses that very `agent wait --status idle` — so `iso-write` and every `--wait`/`deliver` caller is still exposed. The fix belongs in **one tested place in iso-spawn**, with the guiding split that **liveness is universal** (answerable from screen activity, no task knowledge) while **completion is domain-specific** (a generic `idle` proxy, or an artifact the caller injects). Three primitives — `herdr_pane_active`, `wait_recover_settled`, `wait_done` (with a `--done-grep` hook) — carry the universal logic; iso-review keeps only its findings-shape, passed in as `--done-grep`. Full API, migration, and test plan: [docs/superpowers/specs/2026-05-29-spawn-lifecycle-primitives-design.md](../superpowers/specs/2026-05-29-spawn-lifecycle-primitives-design.md).

## Considered Options

- **Leave the logic in each skill** (rejected) — iso-review already carries a duplicate `herdr_agent_status` fallback that it self-documents as drift-prone; N copies of a subtle poll loop is N chances to diverge, and `iso-write` gets none of the robustness.
- **Move *everything* down, including the findings test** (rejected) — would force iso-spawn to know what "findings" are, coupling the shared layer to one consumer's artifact. The universal/domain split avoids this: the caller injects its done-test via `--done-grep`.
- **Three shared primitives + a `--done-grep` hook** (chosen) — one tested copy of liveness/settle-recover/wait-with-reason; `deliver` swaps its bare `agent wait` for `wait_done` so all callers inherit the fixes; iso-review collapses to calling the primitives. With the grain of the existing downward dependency (iso-review/iso-write already source `herdr.sh` and call `spawn.sh recover`).

## Consequences

`wait_done` is kill-agnostic (returns a reason code; disposal stays the caller's). Idle-grace is conditional on `--done-grep` so generic `deliver` calls don't pay a ~30s tax. Liveness is cksum-change-primary (the `esc to interrupt` marker proved unreliable on claude), with one accepted blind spot — a spinner-animating hang is caught only by the timeout ceiling. `iso-spawn/*` is presently another agent's working tree, so this is a hand-off: the spec prescribes no iso-spawn edits from the iso-review line of work; the iso-spawn owner implements it tests-first, keeping `tests/run.sh` green before migrating `deliver` and then `drive.sh`.
